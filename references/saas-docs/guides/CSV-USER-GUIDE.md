# FoundersDirect CSV活用ガイド

**バージョン:** v1.1
**最終更新:** 2025-12-02
**対応フェーズ:** Phase 14.35

> AIを使ってCSVを生成し、FoundersDirectにインポートする方法

---

## はじめに

FoundersDirectでは、**外部AI（ChatGPT / Claude）**を使ってCSVデータを生成し、各タブにインポートできます。

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ あなたの    │     │   AI       │     │ Founders    │
│ ビジネス情報 │ → │ GPT/Claude │ → │ Direct      │
└─────────────┘     └─────────────┘     └─────────────┘
      入力              CSV生成           インポート
```

**メリット:**
- 数分で戦略立案・計画作成が完了
- AIが整ったフォーマットで出力
- そのままインポートして即利用

## 基本の流れ

### ステップ1: プロンプトを準備

下記の「タブ別プロンプト集」から使いたいものをコピーします。

### ステップ2: AIに依頼

ChatGPTまたはClaudeを開き、コピーしたプロンプトを貼り付けます。
【】内を自分のビジネス情報に書き換えてください。

### ステップ3: CSVを保存

AIが出力したCSVをコピーし、テキストエディタに貼り付けて保存します。
- ファイル名: 任意（例: `mvv.csv`）
- 文字コード: **UTF-8**
- 拡張子: **.csv**

### ステップ4: インポート

FoundersDirectの該当タブで「インポート」ボタンをクリックし、CSVファイルを選択します。

## タブ別プロンプト集

### 1. MVV（経営理念）を作る

```
以下の情報を元に、MVV（Mission/Vision/Value）をCSV形式で作成してください。

【私のビジネス】
- 事業内容: ここに入力
- ターゲット顧客: ここに入力
- 提供する価値: ここに入力
- 競合との違い: ここに入力

【出力形式】必ずこの形式で出力：
type,content
mission,（ここに出力）
vision,（ここに出力）
value,（ここに出力）

【ルール】
- mission: 誰に何を提供するか、1文50字以内
- vision: 3〜5年後の具体的な目標、100字以内
- value: 大切にする価値観3つを「・」区切り
```

**出力例:**
```csv
type,content
mission,中小企業の営業チームに、顧客との関係を深めるCRMを提供する
vision,2028年に国内中小企業CRM市場でシェアNo.1を獲得し、10万社の営業改革を支援する
value,顧客第一・データドリブン・チームワーク
```

### 2. OKR（目標と成果指標）を作る

#### 2-1. Objective（目標）

```
以下の情報を元に、今期のObjective（目標）をCSV形式で作成してください。

【現状】
- 事業フェーズ: シード / アーリー / グロース から選択
- 今期の最重要テーマ: ここに入力
- 直近の課題: ここに入力
- チーム人数: ○名

【期間】
- 開始日: 2025-01-01
- 終了日: 2025-03-31

【出力形式】必ずこの形式で出力：
title,description,scope,period_start,period_end,is_archived
（Objective名）,（説明）,company,2025-01-01,2025-03-31,false

【ルール】
- 2〜3個のObjectiveを作成
- scope: company（会社全体）/ team（チーム）/ individual（個人）
- 1つは必ずcompanyレベルで
```

#### 2-2. KeyResult（成果指標）

```
以下のObjectiveに対するKeyResult（成果指標）をCSV形式で作成してください。

【Objective一覧】※上で作ったものを貼り付け
- Q1売上基盤の確立: 初期顧客10社を獲得しPMF検証
- 営業プロセスの型化: 再現性のある営業フロー構築

【出力形式】必ずこの形式で出力：
objective_title,title,target_value,current_value,unit,is_qualitative,calc_method
（所属Objective名）,（KR名）,（目標値）,0,（単位）,false,manual

【ルール】
- 各Objectiveに2〜3個のKR
- 数値で測れるものを優先
- unit例: 件、%、円、社、人
```

### 5. Action Map（施策）を作る

#### 5-1. ActionMap

```
以下のKRを達成するためのActionMap（施策）を作成してください。

【達成したいKR】
- KR名: ここに入力
- 目標値: ○○
- 期限: 2025-03-31

【出力形式】必ずこの形式で出力：
title,description,period_start,period_end,is_archived
（ActionMap名）,（説明）,2025-01-01,2025-03-31,false
```

#### 5-2. ActionItem（具体的タスク）

```
以下のActionMapに紐づくActionItem（具体的タスク）を作成してください。

【ActionMap】
- 名前: Q1新規開拓プラン
- 目標: 新規顧客10社獲得

【出力形式】必ずこの形式で出力：
action_map_title,parent_title,title,description,due_date,priority,status
Q1新規開拓プラン,,（タスク名）,（説明）,2025-01-31,high,not_started
Q1新規開拓プラン,（親タスク名）,（子タスク名）,（説明）,2025-01-15,medium,not_started

【ルール】
- 5〜10個のタスクを作成
- 階層構造を活用（親→子）
- priority: high / medium / low
- status: not_started / in_progress / blocked / done
```

### 6. TODO（日次タスク）を作る

```
以下の目標を達成するための日次タスクをCSV形式で作成してください。

【今週の目標】
ここに入力

【出力形式】必ずこの形式で出力：
title,description,suit,status,start_at,duration_minutes,is_habit,elastic_level
（タスク名）,（説明）,spade,not_started,09:00,60,false,
（習慣タスク）,（説明）,heart,not_started,06:00,15,true,ume

【4象限（suit）の意味】
- spade: 緊急かつ重要（すぐやる）
- heart: 重要だが緊急でない（予定に入れる・習慣化）
- diamond: 緊急だが重要でない（任せる・自動化）
- club: 緊急でも重要でもない（未来創造20%タイム）

【習慣タスクの場合】
- is_habit: true
- elastic_level: ume（最小5分）/ take（標準15分）/ matsu（フル30分+）
```

### 9. 既存データを分析してタスクを生成

#### 見込み客リスト → フォロータスク

```
以下の見込み客リストを分析し、優先フォロータスクを生成してください。

【見込み客リスト】※FoundersDirectからエクスポートしたCSVを貼り付け
name,company,status,channel,memo,created_at
田中太郎,株式会社ABC,responded,hp,初回面談済み,2024-12-01
佐藤花子,XYZ商事,negotiating,real,次回訪問予定,2024-11-15

【出力形式】必ずこの形式で出力：

【分析ルール】
- negotiating → spade（商談中は最優先）
```

## よくある質問

### Q: CSVの文字化けが起きる

**A:** 保存時の文字コードを**UTF-8**にしてください。Windowsのメモ帳の場合、保存時に「文字コード」でUTF-8を選択します。

### Q: インポートでエラーが出る

**A:** 以下を確認してください：
- ヘッダー行（1行目）が正しいか
- 必須項目が空になっていないか
- カンマを含む値が `"` で囲まれているか

### Q: AIが違う形式で出力した

**A:** プロンプトの【出力形式】部分を強調して、「必ずこの形式で」と再度依頼してください。
